Data flow
1
Email sources → Gmail label
LinkedIn, Indeed, WTTJ alerts land in mail@pablosorrantia.com → filtered to Incoming Roles via Filters label.
2
n8n pulls emails every 6 hours
Gmail node reads unread emails from label. Schedule trigger: every 6 hours.
3
Claude extracts individual listings
HTTP Request node calls Claude Haiku API. Prompt extracts title, company, location, URL from each email. One item per listing.
4
Notion logs raw listings
Each listing becomes a Notion row with Pipeline Stage: TO EVALUATE. Source detected from URL.
5
Gmail marks emails as read
Prevents duplicate processing on next run. Not yet built.
6
Morning script evaluates TO EVALUATE rows
Fetches full JD for WTTJ/Indeed. Calls Claude API with full candidate profile prompt. Updates Notion with tier, notes, salary, stage. Not yet built.
Notion pipeline stages
TO EVALUATE
Raw listing logged by n8n. Not yet evaluated by Claude.
EVALUATED - APPLY ASAP
Tier 1 or Tier 2 after morning script runs. Ready for action.
EVALUATED - PASS
Red-flagged by Claude. Logged for audit trail, no action needed.
APPLIED
Application submitted. Time, date, and customization logged.
